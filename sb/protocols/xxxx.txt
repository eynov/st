proto_register "{{PROTO_KEY}}" "{{MENU_NAME}}" "{{TRANSPORT}}" \

────────────────────────────────────────────────
1) build_{{PROTO_FILE}} <port>
────────────────────────────────────────────────
要求：
- 参数1是端口
- 所有变量先 local 声明
- 第一步先校验端口是否为空
- 然后调用 `port_valid "$port"` 做合法性校验，非法则 err 并 return 1
- 再做端口冲突检测：
  - 若传输层是 tcp，用 `port_used_tcp "$port"`
  - 若传输层是 udp，用 `port_used_udp "$port"`
  冲突则 err 并 return 1
- 用 read -p 交互式收集必要参数，全部给默认值，允许直接回车
- 如果某字段有推荐默认值，必须直接体现在提示中
- 自动生成凭据时优先使用 openssl rand
- 如果要生成随机字符串，优先生成 URL/JSON 安全、易存储的值
- 如需自签证书：
  - crt 路径：`${CERT_DIR}/{{PROTO_FILE}}_${port}.crt`
  - key 路径：`${CERT_DIR}/{{PROTO_FILE}}_${port}.key`
  - 使用 openssl req -x509 生成
  - 证书 CN 使用域名/SNI
  - 失败必须 err 并 return 1
- 用 `state_next_id` 生成 id
- 用 jq -n 构造 meta JSON
- meta JSON 必须至少包含字段：
  - id
  - protocol
  - port
  - created_at
  - updated_at
  - enabled
- 以及协议所需的全部附加字段
- enabled 默认为 true
- 成功写入：
  `state_set "$id" "$meta"`
- 成功后输出：
  `ok "{{MENU_NAME}} [${id}] 端口 ${port} 已创建"`
- 任一路径失败都必须 err 并 return 1，不能 exit

────────────────────────────────────────────────
2) edit_{{PROTO_FILE}} <id> <payload_json>
────────────────────────────────────────────────
要求：
- 参数1是实例 id
- 参数2是完整 meta JSON 字符串
- 所有变量先 local 声明
- 用 `echo "$payload" | jq -r '.field'` 读取当前值
- 至少支持修改最常用的 2~4 个字段
- read -p 时必须展示当前值
- 回车保留原值
- 如果修改端口，必须重新校验 `port_valid`
- 如果修改端口，必须重新做端口冲突检测
- 如果支持“重新生成密码/密钥/uuid/token”等，必须让用户明确确认
- 将所有变更拼接为 jq filter，例如：
  `.port = 8443 | .server_name = "example.com" | .updated_at = "xxx"`
- 如果没有任何变更：
  info "无变更"
  return 0
- 最终调用：
  `state_patch "$id" "$filter"`
- 成功后输出：
  `ok "{{MENU_NAME}} [${id}] 已更新"`
- 失败必须 err 并 return 1
